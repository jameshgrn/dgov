# LLM Providers

`dgov` talks to one OpenAI-compatible endpoint at a time, for both:

- worker execution at `dgov run` time
- SOP assignment at `dgov compile` time

Provider configuration is therefore repo-level, not per-task.

## Configuration

Three fields in `.dgov/project.toml` control the endpoint:

```toml
[project]
default_agent = "accounts/fireworks/routers/kimi-k2p5-turbo"
llm_base_url = "https://api.fireworks.ai/inference/v1"
llm_api_key_env = "FIREWORKS_API_KEY"
```

| Field | Meaning |
|-------|---------|
| `default_agent` | Model or router name passed in the request body |
| `llm_base_url` | OpenAI-compatible API base URL |
| `llm_api_key_env` | Name of the env var that holds the API key |

Task-level `agent = "..."` overrides only the model/router name. It does not
override the base URL or the API-key env var.

## Fireworks (default)

```toml
[project]
default_agent = "accounts/fireworks/routers/kimi-k2p5-turbo"
llm_base_url = "https://api.fireworks.ai/inference/v1"
llm_api_key_env = "FIREWORKS_API_KEY"
```

```bash
export FIREWORKS_API_KEY=your-key-here
```

## OpenAI

```toml
[project]
default_agent = "gpt-4.1-mini"
llm_base_url = "https://api.openai.com/v1"
llm_api_key_env = "OPENAI_API_KEY"
```

```bash
export OPENAI_API_KEY=your-key-here
```

## OpenRouter

```toml
[project]
default_agent = "anthropic/claude-3.5-sonnet"
llm_base_url = "https://openrouter.ai/api/v1"
llm_api_key_env = "OPENROUTER_API_KEY"
```

```bash
export OPENROUTER_API_KEY=your-key-here
```

Any model OpenRouter exposes is valid in `default_agent`; pick one whose
output schema your worker tools can consume.

## Other OpenAI-compatible endpoints

```toml
[project]
default_agent = "your-model-name"
llm_base_url = "https://your-provider.example.com/v1"
llm_api_key_env = "YOUR_PROVIDER_API_KEY"
```

```bash
export YOUR_PROVIDER_API_KEY=your-key-here
```

## Compile-time behavior

`dgov compile` calls the LLM when SOP bundling is on. The provider config
must be valid at compile time, not only at run time.

To skip the compile-time LLM call:

```bash
dgov compile .dgov/plans/my-plan --dry-run
```

`--dry-run` uses the identity SOP bundler instead of the LLM-backed one.

## Common mistakes

**Wrong env var name.** `.dgov/project.toml` says `llm_api_key_env =
"OPENAI_API_KEY"` but you exported `FIREWORKS_API_KEY`. Both `dgov compile`
and `dgov run` fail at preflight.

**Confusing model name with endpoint.** `default_agent` is the model or
router identifier. It does not select the provider — the endpoint is set by
`llm_base_url`.

**Assuming a task `agent` switches provider.** Per-task `agent` overrides
the model name only. Provider selection is repo-level.

## Recommendation

For most repos:

1. set `default_agent`
2. set `llm_base_url`
3. set `llm_api_key_env`
4. export the matching env var

That is the whole provider story.
