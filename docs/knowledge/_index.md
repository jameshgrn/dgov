# dgov Knowledge Base

The knowledge base is the map of dgov's world. It is for explanations that
need more continuity than inline comments and more narrative shape than the
ledger.

Use it for concepts, architecture, and operating models. Do not use it for
active bugs, rules, debt, or session notes. Those belong in the ledger, the
governor charter, SOPs, or handover files according to their authority level.

The current starting points are:

- `knowledge-pull-architecture`: why articles cite canonical source files
- `sentrux`: architectural sensing and quality gates
- `settlement-flow`: how candidate changes become accepted work
- `failure-shapes`: how known failures map to typed next actions
- `ledger`: durable operational memory
